## P0 核心算法（最高优先）

### P0-1 ⬜ 评分/人数错配（败犬女主太多了！）
- **现象**：app 显示的评分人数/分数与 bgm 网页端不符；拔作岛需卸载重装才正常。
- **抓包根因**：同名作品在 Bangumi 有多类型条目，靠标题匹配会误选：
  - `464376` type=2(动画/TV) total=**27330** ← 正确目标
  - `343241` type=1(**小说**) total=2353
  - `377761` type=1(漫画@comic) total=206
  - `550507` type=2(动画) total=11（第二季）
  - 代理 `bgmapi.anibt.net` 数据可信（拔作岛 9582 已确认）。问题在选择逻辑。
- **方案**：匹配「动画」来源作品时，**先按 Bangumi `type==2` 过滤**（排除小说/漫画/游戏），再按 `popularity`(rating.total) 加权选代表条目；卸载重装即正常 = 旧匹配被缓存，需在重算时覆盖旧 match。
- **涉及**：`BangumiMappers.kt`(透传 type)、`CrossSourceMerge.kt`(representativeOf 加 type 过滤)、`WorkRepositoryImpl.kt`(搜索/backfill/交叉验证)、`Matching.kt`(WorkMatch 增 subjectType?)。
- **验收**：败犬女主太多了！详情显示 ≈8.0 / 27330 人；拔作岛免重装即正确。

### P0-2 ⬜ 口味画像（用户长期口味，二者共用）
- **要点**：
  - 读取用户**评价过**的作品，使用这些作品**本身的 Bangumi tag**（不是用户自定义 tag）。
  - 高分作品 tag → 正向偏好；低分/抛弃作品 tag → 负向偏好。
  - 记录偏好类型/题材/集数/年代/用户评分习惯。
- **方案**：构建 `TasteProfile`（tag 权重表：正向/负向；类型分布；集数偏好；年代分布；评分均值/方差）。数据来源：本地收藏 + Bangumi 同步的评分。
- **验收**：能输出「高分常见 tag Top N」「低分常见 tag Top N」「偏好集数区间」。

### P0-3 ⬜ 口味匹配度（详情页：这部适不适合我）
- **要点**：长期口味为主；命中正向 tag 越多越高、命中负向越多越低；平台评分仅低权辅助；输出可解释；**低数据量显示低置信而非乱给高分**；不考虑「今晚」临时 tag。
- **方案**：`tasteMatchScore(work, profile)` = Σ(命中正向tag权重) − Σ(命中负向tag权重) + 小权重(平台分) ；置信度由用户样本量决定；输出命中/未命中标签解释串。
- **验收**：详情页显示百分比 + 解释（命中你高分常见的「X」「Y」；与低分标签重合少；评分仅辅助）；新用户样本少时显示「低置信」。

### P0-4 ⬜ 今晚看什么（首页：当前心情/场景选最适合一部）
- **要点**：当前选的 tag/心情是**硬筛**（不是加分）；明显冲突作品排除（选恋爱日常不推进击的巨人）；候选池内按长期口味排序；平台评分/人数/热度仅质量兜底；随机性只在高匹配候选内轻微打散；结果不足时提示「是否放宽条件」，不乱推。
- **方案**：先按 tag 硬过滤候选池 → 冲突 tag 黑名单排除 → 长期口味排序 → 质量兜底 → 高分段内轻度 shuffle；候选 < 阈值时返回「放宽条件」状态。
- **验收**：选「恋爱+日常」只出恋爱/日常/校园/轻松类；进击的巨人不出现；结果少时给放宽提示。

### P0-5 ⬜ 口味匹配 AI 分析「置信不足不展示」
- **现象**：经常因置信度不足而不展示，体验差。
- **方案**：降低展示门槛；置信不足时**仍展示本地规则版**结果并标注「低置信/数据较少」，AI 失败时回落本地。
- **验收**：任何情况下详情页都能看到匹配度内容（区分高/低置信样式）。

## P1 数据与同步

### P1-1 ⬜ 详情页 Bangumi 优先
- **现象**：打开是 Jikan 源，进去有 Bangumi 评分但简介/标题全是外语。
- **方案**：详情若存在 Bangumi 词条，标题/简介/tag 自动用 Bangumi 中文内容覆盖 Jikan。
- **验收**：有 bgm 词条的作品详情，标题/简介为中文。

### P1-2 ⬜ 移除「想看」同步删除到 Bangumi
- **现象**：标「想看」会同步到 bgm，但**取消想看**要去网页端删。
- **方案**：app 内移除想看时调用 Bangumi 删除收藏 API（`DELETE/PUT /v0/users/-/collections/{id}`）。
- **验收**：app 内取消想看后，bgm 端同步移除。

### P1-3 ⬜ 口味画像自动更新
- **现象**：每次都要手动更新。
- **方案**：Bangumi 同步完成 或 某作品状态/评分变更后，自动触发画像重算（防抖）。
- **验收**：同步/改状态后画像自动刷新，无需手动按钮。

### P1-4 ⬜ 待补池 ↔ 吃灰馆 移动一致性
- **现象**：待补池移入吃灰馆后会从待补池消失，直到移出；吃灰馆也是多选后移出。
- **方案**：明确两池为互斥归属（移动而非复制）；统一多选移动交互；移动后双方列表即时刷新。
- **验收**：移动后来源消失、目标出现；多选移动可用且一致。

## P2 UI / UX

### P2-1 ⬜ 封面大图缩放 + 拖动
- **方案**：大图查看器支持双指/双击缩放 + 放大后平移（pointerInput + graphicsLayer transform）。
- **验收**：可缩放、放大后可拖动浏览。

### P2-2 ⬜ 发现-榜单分页加载（>60）
- **现象**：只显示前 60 名，到底不加载。
- **方案**：LazyColumn 触底分页加载后续；保留已加载项。
- **验收**：滑到底自动加载第 61+ 名。

### P2-3 ⬜ 榜单本地缓存
- **现象**：每次点开都重新加载。
- **方案**：榜单结果按 key+时间戳缓存本地（Room/DataStore），命中缓存先展示，变更/过期才重载。
- **验收**：再次进入秒开，后台校验变更才刷新。

### P2-4 ⬜ 详情页「适合心情」→ Bangumi 原生 tag
- **方案**：详情页展示 Bangumi 该作品 tag（替换现「适合心情」）。
- **验收**：详情显示 bgm tag 列表。

### P2-5 ⬜ 今晚看什么「现在的心情」→ tag 筛选（改名）
- **方案**：将「现在的心情」改为 tag 筛选入口；建议命名：**「今晚想看的类型」/「场景标签」/「氛围筛选」**（待定名）。
- **验收**：UI 文案与交互改为按 tag 筛选。

### P2-6 ⬜ 发现-筛选 题材不可用
- **现象**：题材筛选选什么都没内容。
- **方案**：排查题材枚举与数据 tag/genre 映射不一致；修正过滤查询。
- **验收**：选任一题材有正确结果。

### P2-7 ⬜ 发现-评分差异 内容太少
- **方案**：丰富评分差异维度（多平台分差、人数、排名、跨源标准差等）与展示。
- **验收**：评分差异页信息更充实。

### P2-8 ⬜ 主页「今日状态」选择框
- **现象**：意义不明、感觉没用。
- **方案**：评估移除或重新定义（如改为「今日心情→今晚看什么」入口）。需与用户确认方向。
- **验收**：按确认方案处理。

## P3 AI 提示词

### P3-1 ⬜ 评论摘要 ↔ 无剧透雷达 提示词调换/重定义
- **理解**：用户认为「无剧透评论雷达」应输出**亮点/雷点 + 前期制作 + 节奏（快/缓）**，而非现在的内容；与「评论摘要」对调或重写。
- **方案**：重写两个 AI 任务的 prompt 与输出结构：雷达=亮点雷点+制作水准+节奏评价（无剧透）；摘要=社区评论聚合概述。
- **涉及**：`AiTaskSpecs.kt`、`AiTask.kt`、`AiOutputs.kt`、详情页展示。
- **验收**：雷达显示亮点/雷点/制作/节奏；摘要显示评论聚合。
